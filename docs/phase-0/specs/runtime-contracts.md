# Phase 0 Runtime Contracts

## Contract Rules

- Runtime Core owns these contracts.
- LangChain cannot mutate these objects directly.
- All persisted contract objects include `version`.
- All timestamps use UTC ISO-8601 strings.
- IDs are opaque strings generated by runtime.

## Session

```python
class Session:
    session_id: str
    workspace_root: str
    status: str
    approval_mode: str
    active_run_id: str | None
    artifact_root: str
    config_snapshot: dict
    latest_checkpoint_id: str | None
    created_at: str
    updated_at: str
    error_summary: str | None
    version: int
```

Allowed `status` values:

- `running`
- `completed`
- `failed`
- `interrupted` is reserved for Phase 2 and later.

Phase 0 creates one prompt run per one-shot session and one long-lived prompt run per REPL session.

## Run

```python
class Run:
    run_id: str
    session_id: str
    parent_run_id: str | None
    run_type: str
    status: str
    active_skills: list[str]
    latest_checkpoint_id: str | None
    context_snapshot_id: str | None
    created_at: str
    updated_at: str
    error_summary: str | None
    version: int
```

Phase 0 only supports `run_type = "prompt"`.

`context_snapshot_id` is always `None` in Phase 0. It is reserved for Phase 1 ContextManager work.

Allowed `status` values:

- `running`
- `completed`
- `failed`
- `interrupted` is reserved for Phase 2 and later.

Phase 0 persisted runs use `running`, `completed`, and `failed`.

## RunEvent

```python
class RunEvent:
    event_id: str
    timestamp: str
    session_id: str
    run_id: str
    step_id: str | None
    kind: str
    payload: dict
    version: int
```

Phase 0 event kinds:

- `session_started`
- `session_completed`
- `session_failed`
- `run_started`
- `run_completed`
- `run_failed`
- `user_message`
- `assistant_message`
- `model_call_started`
- `model_call_completed`
- `model_call_failed`
- `tool_call_started`
- `tool_call_completed`
- `tool_call_denied`
- `tool_call_failed`
- `checkpoint_written`
- `artifact_registered`

Phase 0 event payloads may include inline model response content and tool output
only when the serialized content is at or below 16 KiB. Larger model response
content and tool output are stored as artifacts and referenced by artifact id.

`model_call_completed` payload includes usage, duration, model response content
or redacted artifact references, and normalized tool calls when the model
requested tools.

`tool_call_completed` payload includes the standard audit fields plus the
standardized `ToolResult` payload as `result`.

## Checkpoint

```python
class Checkpoint:
    checkpoint_id: str
    session_id: str
    run_id: str
    kind: str
    state: dict
    summary: str | None
    created_at: str
    version: int
```

Phase 0 `kind` values:

- `turn`: written after a successful one-shot response or REPL turn.
- `terminal`: written when a run/session exits successfully, including `/exit`.
- `error`: written before a failed terminal state when enough runtime state exists to persist.

Phase 0 checkpoint state includes:

- session status
- run status
- prompt turn counter
- latest model response metadata
- latest artifact ids
- latest error summary

Checkpoint must not store long stdout, full trace, binary content, or large model/tool output. Large content goes to ArtifactStore.

## Artifact

```python
class Artifact:
    artifact_id: str
    session_id: str
    run_id: str | None
    relative_path: str
    artifact_type: str
    metadata: dict
    created_at: str
    version: int
```

Phase 0 artifact types:

- `image`
- `rdc`
- `text`

## ToolResult

```python
class ToolResult:
    status: str
    output: str | dict | None
    error: dict | None
    artifacts: list[str]
    metadata: dict
    redacted_output: str | None
```

`redacted_output` is used when output is too large for inline display. Phase 0 treats outputs larger than 16 KiB as large and stores the full content as an artifact.

Allowed `status` values:

- `ok`
- `error`
- `denied`
- `timeout`
- `cancelled`

## Error Classification

Phase 0 uses the shared v1 error classes:

- `user_error`
- `config_error`
- `policy_denied`
- `tool_error`
- `model_error`
- `workflow_error`
- `internal_error`
- `timeout`
- `cancelled`

`workflow_error` is reserved for future compatibility and should not be emitted by Phase 0 normal paths.

## State Machine

```text
running -> completed
running -> failed
```

Phase 0 does not persist long-lived `interrupted` sessions because `/resume` is not available. Ctrl+C or mid-call cancellation is recorded as `failed` with error class `cancelled`, followed by an `error` checkpoint when possible.

## Active Session Ownership

- Session creation must check whether `workspace_root` already has an active session.
- Phase 0 active status is `running`.
- If an active session exists for the same `workspace_root`, new session creation fails with `user_error`.
- Ownership is recorded in SQLite and may be mirrored with a local lock or anchor file for fast detection.
- Ownership is released when session reaches `completed` or `failed`.

Phase 2 may reintroduce `interrupted` as an active ownership state together with `/resume`.
