# Phase 4 Deployment Specification

## Boundary

Phase 4 standardizes a package/deployment smoke path for `debug-agent`.

The deployment smoke verifies that the existing Python package metadata and
console script can be built, installed into a uv tool environment, and invoked
outside the source checkout.

This spec does not change runtime workspace semantics, configuration sources,
path policy, active ownership, or skill discovery.

## Package Entry Point

`pyproject.toml` declares:

```toml
[project.scripts]
debug-agent = "debug_agent.cli.main:main"
```

Phase 4 uses this entry point as the installed CLI command.

## Recommended Install Forms

Local source install for quick verification:

```bash
uv tool install /Users/xinzhu/Workspace/MyAgent
debug-agent --help
```

Wheel install for reproducible deployment smoke:

```bash
uv build
uv tool install --force dist/debug_agent-0.1.0-py3-none-any.whl
debug-agent --help
```

The wheel under `dist/` is generated by `uv build`. It is not committed to the
source repository.

## Runtime Workspace Semantics

Installed `debug-agent` may be invoked from any directory, but runtime behavior
is still rooted in the current workspace:

```bash
cd /path/to/target/workspace
debug-agent
```

Workspace-rooted behavior includes:

- `.sessions/` runtime data.
- project `.debug-agent/skills/` discovery.
- workspace active session ownership.
- path policy.
- artifact paths.
- tool file access boundaries.

The tool installation directory is not the runtime workspace.

## Configuration Sources

Phase 4 deployment does not change config sources.

Runtime config remains:

```text
~/.debug-agent/config.toml
~/.debug-agent/agent.toml
```

Project-local `config.toml` remains unsupported.

The deployment smoke must not require copying source-tree `.venv` state.

## Standard Smoke

The canonical package smoke verifies:

1. package builds.
2. package installs through `uv tool install`.
3. installed console script runs outside the source checkout.

Minimum command sequence:

```bash
uv build
uv tool install --force dist/debug_agent-0.1.0-py3-none-any.whl
mkdir -p /tmp/debug-agent-smoke
cd /tmp/debug-agent-smoke
debug-agent --help
```

The exact wheel filename may change with project version. Operations may use the
single generated wheel path from `dist/`.

## Optional Runtime Smoke

When the target machine has valid model configuration, a broader manual smoke
may run:

```bash
cd /tmp/debug-agent-smoke
debug-agent -p "hello"
debug-agent status <session_id>
debug-agent trace <session_id>
```

This optional runtime smoke verifies that an installed command can create
`.sessions/`, read user-level config, persist runtime data, and generate trace
output outside the source checkout.

It is not required for ordinary package smoke because it depends on local model
configuration.

## External Requirements

Deployment targets need:

- Python compatible with `requires-python`.
- `uv`.
- runtime dependencies declared in `pyproject.toml`.
- external tools required by selected runtime features, such as `rg` for
  `search_text`.
- RenderDoc/`rdc` only for RenderDoc smoke, not for `debug-agent --help`.

Absence of optional external tools is not a package installation failure unless
the specific smoke scenario uses that tool.
